# Threat Handling

Threat handling is evidence-based and transparent.

## Flow

1. Collect file, process, behavior, network, and integrity evidence.
2. Score each layer independently.
3. Generate an incident report when a threshold is crossed.
4. Quarantine by default when confidence is high and policy allows it.
5. Ask the user before destructive actions unless strict policy explicitly enables automatic action.
6. Explain every detection reason and confidence score.
7. Support false-positive review, restore and exclude, and local allowlisting.

AI can explain and summarize evidence, but it must not block by itself.
